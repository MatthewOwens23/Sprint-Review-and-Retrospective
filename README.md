#  Reflection: Agile and Scrum Practices
#  Interpreting User Needs and User Stories

I interpret user needs by focusing on what the user is trying to accomplish rather than just the technical requirements. Instead of jumping straight into coding, I break requirements into smaller, manageable goals. Creating user stories helps with this because they frame each feature from the user’s perspective, typically in the format As a user, I want ___ so that ___. This approach makes it easier to understand the purpose behind each feature and ensures the program delivers real value. User stories also help prioritize tasks and guide development by keeping the focus on usability and functionality rather than just implementation.

# My Approach to Developing Programs and Agile Practices

When developing programs, I use an iterative approach by building small pieces of functionality, testing them, and improving them over time. Throughout this course, I applied Agile practices such as sprint planning, sprint reviews, and retrospectives to organize my workflow. These practices helped keep development structured while still allowing flexibility for changes based on feedback. In the future, I plan to continue using Agile methods like maintaining a backlog, working in short sprints, and regularly reviewing progress. This approach supports continuous improvement and ensures that projects remain aligned with user needs.

# Being a Good Team Member in Software Development

Being a good team member in software development means communicating clearly, collaborating effectively, and being open to feedback. It also involves taking responsibility for assigned tasks while supporting teammates when needed. A strong team member contributes to discussions, respects different perspectives, and works toward shared goals. In Agile environments, teamwork is especially important because progress depends on coordination and transparency. By staying engaged and adaptable, I can contribute to a productive and positive development environment.
